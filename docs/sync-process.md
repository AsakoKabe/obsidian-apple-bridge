# Процесс синхронизации

Apple Bridge выполняет двустороннюю синхронизацию между вашим Obsidian-хранилищем и Apple-приложениями: Calendar, Reminders, Notes и Contacts. Все взаимодействие с Apple-приложениями происходит через JXA (JavaScript for Automation) скрипты, запускаемые через `/usr/bin/osascript`.

## Общий принцип работы

Синхронизация запускается одним из трёх способов:

1. **Автоматически** — по таймеру (настраиваемый интервал, по умолчанию 15 минут)
2. **Вручную** — через команду `Sync Apple Apps now` (Ctrl/Cmd+P)
3. **Кнопка** — через иконку обновления в боковой панели Obsidian

При запуске плагин параллельно (`Promise.all`) синхронизирует все включённые модули: Calendar, Reminders, Notes и Contacts. Каждый модуль можно включить/выключить независимо в настройках.

## Хранение состояния синхронизации

Каждый модуль хранит своё состояние (sync state) в `data.json` плагина под уникальным ключом:

| Модуль    | Ключ                    | Содержимое                                              |
| --------- | ----------------------- | ------------------------------------------------------- |
| Calendar  | `calendar-sync-state`   | Словарь событий по Apple ID с последним состоянием      |
| Reminders | `reminders-sync-state`  | Словарь напоминаний по Apple ID с последним состоянием   |
| Notes     | `notes-sync-state`      | Словарь заметок по Apple ID с путём в хранилище         |
| Contacts  | `contacts-sync-state`   | Словарь контактов по Apple ID с путём в хранилище       |

Состояние позволяет отслеживать, какие данные изменились с последней синхронизации — как на стороне Apple, так и на стороне хранилища.

## Calendar (Двусторонняя синхронизация)

### Шаги синхронизации

1. **Получение событий из Apple Calendar** — запрос всех событий за текущий день через JXA
2. **Загрузка sync state** — прошлое состояние из `data.json`
3. **Создание/открытие daily note** — файл `YYYY-MM-DD.md` в настроенной папке
4. **Парсинг событий из заметки** — поиск секции `## Calendar Events` и парсинг строк формата:
   ```
   - [ ] HH:MM - HH:MM Название события 📍 Локация [id:apple-event-id]
   ```
5. **Пуш локальных событий в Apple** — если событие в заметке не имеет `[id:...]`, оно создаётся в Apple Calendar
6. **Детекция локальных изменений** — сравнение текущих строк заметки с сохранённым состоянием (title, location, time)
7. **Мерж изменений с разрешением конфликтов** — при наличии изменений и на стороне Apple, и в заметке, применяется выбранная стратегия
8. **Пуш оставшихся локальных правок** — изменения, не конфликтующие с Apple, отправляются в Apple Calendar
9. **Запись объединённых событий обратно в заметку**
10. **Сохранение нового sync state**

### Формат записи событий в daily note

```markdown
## Calendar Events

- [ ] 09:00 - 10:00 Standup 📍 Zoom [id:E1234]
- [ ] all-day Conference [id:E5678]
- [ ] 14:00 - 15:00 Мой локальный ивент
```

- Строки с `[id:...]` — уже синхронизированные с Apple
- Строки без `[id:...]` — локальные, будут созданы в Apple при следующей синхронизации
- Пользовательские строки без формата событий сохраняются как есть

### Создание события через UI

Команда `Create Calendar Event` открывает модальное окно с полями: название, дата, время начала/окончания, all-day, локация, заметки. После создания событие записывается одновременно в Apple Calendar и в daily note, а sync state обновляется.

## Reminders (Двусторонняя синхронизация)

### Шаги синхронизации

1. **Получение незавершённых напоминаний из Apple Reminders**
2. **Загрузка sync state**
3. **Создание/открытие daily note**
4. **Парсинг напоминаний из заметки** — секция `## Reminders`:
   ```
   - [ ] Купить молоко 📅 2026-04-10 [rid:R1234]
   - [x] Отправить письмо [rid:R5678]
   ```
5. **Пуш локальных напоминаний** — строки без `[rid:...]` создаются в Apple Reminders
6. **Детекция локальных изменений** — title, isCompleted, dueDate
7. **Мерж с разрешением конфликтов**
8. **Пуш оставшихся локальных правок**
9. **Запись объединённых напоминаний обратно в заметку**
10. **Сохранение нового sync state**

### Формат записи напоминаний

```markdown
## Reminders

- [ ] Купить молоко 📅 2026-04-10 [rid:R1234]
- [x] Завершённое напоминание [rid:R5678]
- [ ] Локальное напоминание
```

- `[x]` / `[ ]` — статус выполнения (синхронизируется с Apple)
- `📅 YYYY-MM-DD` — срок выполнения (опционально)
- `[rid:...]` — идентификатор Apple Reminders

## Notes (Односторонняя: Apple → Obsidian)

### Шаги синхронизации

1. **Получение всех заметок из Apple Notes** через JXA
2. **Загрузка sync state**
3. **Обработка каждой заметки**:
   - Сравнение `modificationDate` с сохранённым состоянием
   - Если заметка не изменялась — пропуск
   - Если новая или изменённая — конвертация HTML в Markdown и запись в хранилище
   - Если заметка переименована/перемещена — удаление старого файла, создание нового
4. **Удаление удалённых заметок** — если Apple Note больше не существует, файл удаляется из хранилища
5. **Сохранение нового sync state**

### Формат импортированных заметок

Каждая заметка сохраняется как отдельный markdown-файл с frontmatter:

```markdown
---
apple_note_id: "x-coredata://..."
created: 2026-04-01T10:00:00Z
modified: 2026-04-10T12:00:00Z
folder: "Notes/Projects"
---

Содержимое заметки в формате Markdown...
```

Файлы размещаются в папке, повторяющей структуру папок Apple Notes (по умолчанию `Apple Notes/`).

### Конвертация HTML → Markdown

Плагин конвертирует HTML-контент Apple Notes в Markdown, поддерживая:

- Заголовки (h1–h6)
- Жирный, курсив, зачёркнутый текст
- Ссылки и изображения
- Списки (маркированные, нумерованные)
- Чеклисты Apple Notes
- Таблицы
- Блоки кода
- Цитаты
- Горизонтальные линии

## Contacts (Односторонняя: Apple → Obsidian)

### Шаги синхронизации

1. **Получение всех контактов из Apple Contacts** через JXA
2. **Загрузка sync state**
3. **Обработка каждого контакта**:
   - Сравнение `modificationDate` с сохранённым состоянием
   - Если не изменялся — пропуск
   - Если новый или изменённый — запись/обновление файла в хранилище
   - Если контакт переименован — удаление старого файла, создание нового
4. **Удаление удалённых контактов** — если контакт больше не существует в Apple Contacts, файл удаляется
5. **Сохранение нового sync state**

### Формат карточки контакта

```markdown
---
apple_contact_id: "A1234"
modified: 2026-04-10T12:00:00Z
birthday: 1990-05-15
emails:
  - label: "work"
    value: "john@example.com"
phones:
  - label: "mobile"
    value: "+7 999 123 4567"
---

# John Doe

Developer at Acme Corp

## Contact

- work: john@example.com
- mobile: +7 999 123 4567

## Addresses

- home: 123 Main St, Moscow, Russia

## Notes

Личные заметки о контакте...
```

Файлы размещаются в настроенной папке (по умолчанию `People/`).

## Разрешение конфликтов

Конфликт возникает, когда и Apple-данные, и данные в заметке изменились с момента последней синхронизации. Применяется к Calendar и Reminders (двусторонняя синхронизация).

| Стратегия       | Описание                                                                     |
| --------------- | ---------------------------------------------------------------------------- |
| `remote-wins`   | Данные из Apple перезаписывают локальные изменения (по умолчанию)             |
| `local-wins`    | Данные из хранилища отправляются в Apple, перезаписывая удалённые изменения  |
| `most-recent`   | Побеждает та сторона, чьи изменения были сделаны позднее                     |

Стратегия выбирается глобально в настройках плагина и применяется ко всем модулям с двусторонней синхронизацией.

## Настраиваемые папки

Каждый модуль пишет данные в настраиваемую папку хранилища:

| Модуль    | Настройка          | По умолчанию   | Формат файлов                  |
| --------- | ------------------ | -------------- | ------------------------------ |
| Calendar  | `calendarFolder`   | _(корень)_     | `YYYY-MM-DD.md` (daily notes)  |
| Reminders | `remindersFolder`  | _(корень)_     | `YYYY-MM-DD.md` (daily notes)  |
| Notes     | `notesFolder`      | `Apple Notes`  | `Папка/Название заметки.md`    |
| Contacts  | `contactsFolder`   | `People`       | `Имя Фамилия.md`              |

## Ограничения

- **macOS only** — плагин работает только на macOS, так как использует JXA для взаимодействия с Apple-приложениями
- **Calendar и Reminders** — синхронизируются только события/напоминания за текущий день
- **Notes и Contacts** — синхронизация только в одну сторону (Apple → Obsidian), изменения в Obsidian не отправляются обратно в Apple
- **Требуются разрешения** — macOS может запросить разрешение на доступ к Calendar, Reminders, Contacts и Notes при первом запуске
